    ↓

    ↓

    ↓
Готово к использованию

При setActiveCell():
    • Сохраняется ссылка на элемент
    • Сохраняются координаты { row, col }
    • Добавляется класс 'focus-manager-active'

При потере фокуса:
    • MutationObserver обнаруживает изменения
    • Или фоновая проверка обнаруживает потерю
    • Вызывается restoreFocus()
    • Ячейка скроллится в view
    • Фокус восстанавливается на element.focus()

💡 СОВЕТЫ ДЛЯ ИСПОЛЬЗОВАНИЯ
═══════════════════════════════════════════════════════════════════════════════

1. ВСЕГДА вызывайте restoreFocus() после операций с DOM:
   • После очистки кэша
   • После re-render
   • После создания/удаления элементов

2. Используйте getCellByCoords callback для получения ячейки:
   • Это основной способ найти ячейку после re-render
   • Убедитесь что callback работает правильно

3. Поддерживайте data-row и data-col атрибуты:
   • FocusManager полагается на эти атрибуты
   • Не удаляйте их при обновлении элементов

4. Включайте отладку при проблемах:
   • FocusManager.setDebug(true)
   • Посмотрите логи в консоли
   • Проверьте что getCellByCoords возвращает элемент
